## One-Liner

Practical guide to building LLM agents using simple, composable patterns—covering workflows (prompt chaining, routing, parallelization) vs autonomous agents.

## Key Points

- Distinguishes **workflows** (predefined code paths) from **agents** (LLMs that dynamically direct their own processes)
- Start simple: only add agentic complexity when it demonstrably improves outcomes
- Five key workflow patterns: Prompt chaining, Routing, Parallelization, Orchestrator-workers, Evaluator-optimizer
- Core agent principles: simplicity, transparency, and thorough tool documentation (ACI)
- Agents trade latency/cost for better task performance on open-ended problems
